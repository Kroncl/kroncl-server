# Kroncl | Server-side

Enterprise‑grade ERP‑система для малого бизнеса. Мультитенантная архитектура с физической изоляцией данных, встроенной админкой, метриками и ролевой моделью.

![Да, это прекрасно](https://kroncl-client.vercel.app/images/mock-ups/company-overview-cut.png)

## Стек

- **Go 1.24+**
- **PostgreSQL 16+** (изоляция тенантов через схемы)
- **pgx** – драйвер и пул соединений
- **chi** – роутер, middleware
- **golang‑migrate/migrate** – миграции
- **MinIO** – объектное хранилище
- **Prometheus + Grafana** – мониторинг
- **Caddy** – веб‑сервер, автоматические SSL‑сертификаты

## Архитектура

- **Монолит с чёткими границами** – код разделён на доменные пакеты (accounts, companies, pricing, admin...). Каждый пакет самодостаточен и имеет чёткую зону ответственности.
- **Мультитенантность** – каждая организация получает отдельную схему БД (company_*). Подключение к схеме кэшируется в пуле.
- **Ролевая модель (RBAC + тарифы)** – разрешения привязаны к тарифному плану, должности и переопределениям. Все действия логируются.
- **Административная панель** – уровни доступа 1–5, ключевое слово для критичных операций, мониторинг БД, управление тикетами, заявками на партнёрство.
- **Наблюдаемость** – два воркера собирают метрики: нагрузка на БД (pg_stat_database) и клиентские метрики (аккаунты, компании, платежи). История хранится в БД, отдаётся через админку.

## Ключевые возможности

- **Управление сотрудниками (HRM)** – карточки сотрудников без обязательной регистрации, должности с правами доступа, привязка аккаунта к сотруднику.
- **Финансы (FM)** – операции, категории, долговые обязательства, сторнирование, аналитика.
- **Каталог и склад (WM)** – товары/услуги, партионный/поштучный учёт, поставки/отгрузки, остатки.
- **Клиентская база (CRM)** – клиенты, источники привлечения, аналитика по каналам.
- **Сделки (DM)** – канбан, статусы, типы, состав заказа из каталога, интеграция с финансами.
- **Техподдержка** – тикеты внутри компаний, администрирование тикетов, обмен сообщениями.
- **Партнёрская программа** – заявки от потенциальных партнёров, модерация, смена статуса.

## Роутинг и middleware

- **chi** с группировкой роутов по модулям.
- **JWT‑авторизация** (access/refresh токены, httpOnly‑куки).
- **Админские мидлвари** – RequireAdmin, RequireAdminLevel, RequireAdminKeyword.
- **Permission‑мидлварь** – проверка конкретного разрешения с учётом тарифа, роли и переопределений.
- **Тенантный пул** – компания прокидывается в контекст, из него достаётся пул соединений к схеме организации.
- **Лимиты** – httprate для публичных и приватных маршрутов.

## Миграции

- **golang‑migrate** + кастомная обёртка.
- Две директории: `migrations/public` (общая схема) и `migrations/tenant` (схемы компаний).
- CLI‑утилита (`cmd/migrate`) с поддержкой флагов:
  - `--type public|tenant|all-tenants`
  - `--schema` – имя конкретной схемы.
  - `--cmd up|down|version|force|create`
- При создании новой компании асинхронно накатываются tenant‑миграции.

## Метрики

- **DB‑воркер** (interval 60s) – собирает размер БД, активные схемы, таблицы, индексы, статистику из pg_stat_database.
- **Clientele‑воркер** (interval 60s) – снимает срезы аккаунтов, компаний, связей и транзакций.
- История метрик хранится в таблицах `metrics_db_history` и `metrics_clientele_history`. Данные отдаются через админку с фильтром по дате.

## Запуск

**Обязательные условия:**
- Установлен Docker и Docker Compose.
- Создан файл `.env` по образцу `.env.example` (все переменные окружения, включая `FIRST_ADMIN_EMAIL` и `ADMIN_KEYWORD`).

**Продакшен / полный стек (с Caddy):**
```bash
docker compose up -d
```

**Локальная разработка (без Caddy, проброс порта API напрямую):**
```bash
docker compose -f docker-compose.local.yml up -d
```

После этого:
- API доступен на `http://localhost:8080`
- Grafana – `http://localhost:3030`
- MinIO Console – `http://localhost:9001`
- pgAdmin – `http://localhost:5050`

**Миграции:**
```bash
# Публичная схема (up)
go run cmd/migrate/main.go --type public --cmd up

# Все tenant‑схемы (up)
go run cmd/migrate/main.go --type all-tenants --cmd up

# Конкретная схема (tenant)
go run cmd/migrate/main.go --type tenant --schema company_xxx --cmd up
```
